# EduTrack — Academic Progress Portal
Flutter app with Firebase backend. Role-based portal for Teachers, Students, and Parents.

## Features
- 🎓 Role-based login (Teacher / Student / Parent)
- 📋 Live lesson progress logs from Firebase
- 📊 Subject progress bars & stats dashboard
- 📝 Teachers can add/edit lesson entries
- 📈 Exam results per term
- 🔔 Auto-generated alerts from observations
- 👥 Manage users (add students, parents, teachers)

## Setup

### 1. Firebase Setup
1. Go to [Firebase Console](https://console.firebase.google.com)
2. Create a new project (e.g. `edutrack-portal`)
3. Enable **Authentication** → Email/Password
4. Enable **Firestore Database** → Start in test mode
5. Go to **Project Settings** → Add app (Flutter/Android/iOS/Web)

### 2. FlutterFire CLI
```bash
dart pub global activate flutterfire_cli
flutterfire configure
```
This auto-generates `lib/firebase_options.dart` with your real config.

### 3. Migrate Your Excel Data to Firebase
```bash
cd edutrack
pip install firebase-admin openpyxl pandas
# Download serviceAccountKey.json from Firebase Console → Project Settings → Service accounts
# Export your Google Sheet as Academic_Progress_Report.xlsx
python migrate_to_firebase.py
```

### 4. Run the App
```bash
flutter pub get
flutter run
```

## Firestore Structure
```
users/
  {uid}/
    email, name, role, linkedStudent, class

lessonLogs/
  {docId}/
    studentName, date, subject, teacher, topic,
    progress, homework, observations, followUp, class

examResults/
  {docId}/
    studentName, term, subject, score, feedback
```

## Adding More Students
1. Export each student's sheet as Excel
2. Update `migrate_to_firebase.py` with the new student name
3. Run the migration script again
4. Add a new user via the app (Teacher → Manage Users)
